# 13장 과제: 웹 애플리케이션과 영속성 관리

## 📖 학습 내용 요약 (13장)
- **트랜잭션 범위 영속성 컨텍스트**: 트랜잭션 시작과 동시에 생성, 종료와 동시에 소멸.
- **지연 로딩과 준영속 상태**: 트랜잭션 종료 시 엔티티는 준영속 상태가 되어 지연 로딩 불가능 (`LazyInitializationException`).
- **변경 감지**: 트랜잭션 내에서 영속 상태인 엔티티의 필드 수정 시 `save()` 없이도 자동 업데이트.
- **해결책**:
  - `fetch join`: 필요한 데이터를 한 번에 가져오기.
  - `OSIV (Open Session In View)`: 요청부터 응답까지 컨텍스트 유지 (장단점 고려 필요).

---

## 🛠️ 과제 요구사항
**목표**: `MemberService`와 `MemberRepository`를 수정하여 트랜잭션 범위에 따른 동작을 확인한다.

### 1. 변경 감지 테스트 (Scenario A)
- `changeName`: `@Transactional`이 있을 때 `save()` 없이 업데이트 확인.
- `changeNameWithoutTransaction`: `@Transactional`이 없을 때 업데이트가 반영되지 않음 확인.

### 2. 지연 로딩 테스트 (Scenario B)
- `findMember`: 기본 조회 후 트랜잭션 밖에서 지연 로딩 시 에러 확인.
- `findMemberWithTeam`: **fetch join**을 통해 트랜잭션 밖에서도 팀 정보 접근 가능하게 구현.
