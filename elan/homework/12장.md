# 12장 과제: 스프링 데이터 JPA

## 📖 학습 내용 요약 (12장)
- **스프링 데이터 JPA**: 인터페이스 선언만으로 CRUD를 자동화하는 프로젝트.
- **주요 기능**:
  - `JpaRepository` 상속: 기본 CRUD 메서드 자동 제공.
  - **쿼리 메서드**: 메서드 이름 규칙에 따라 JPQL 자동 생성 (`findByUsername`).
  - **@Query**: 직접 JPQL 또는 네이티브 쿼리 작성 가능.
  - **사용자 정의 리포지토리**: 복잡한 로직은 별도 인터페이스와 구현체로 확장 가능.

---

## 🛠️ 과제 요구사항
**목표**: `MemberRepository` 인터페이스를 완성하여 테스트를 통과시킨다.

### 1. 쿼리 메서드 구현 (Q1)
- `findByName(String name)`: 이름 일치 회원 조회.
- `findByNameAndAgeGreaterThan(String name, int age)`: 이름 일치 + 나이 초과 조회.
- `findByNameStartingWith(String prefix)`: 특정 문자열로 시작하는 이름 조회.

### 2. @Query 사용 (Q2)
- `findByTeamName(@Param("teamName") String teamName)`: 특정 팀 이름에 소속된 회원 조회 (Join 필요).
- `findByNameContaining(@Param("keyword") String keyword)`: 이름에 키워드 포함 조회 (LIKE 사용).
