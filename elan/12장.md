## 12장 스프링 데이터 JPA

### 12.1 스프링 데이터 JPA 소개

**개념**
- JPA를 스프링 프레임워크에서 편리하게 사용할 수 있도록 지원하는 프로젝트
- 데이터 접근 계층에서 반복되는 CRUD 코드의 문제를 해결

**핵심 이점**
- CRUD를 처리하기 위한 공통 인터페이스 제공
- 인터페이스만 작성하면 구현 클래스 없이 개발 완료 가능
- 실행 시점에 스프링이 동적으로 구현 객체 생성 후 주입

### 12.2 스프링 데이터 JPA 설정

**의존성 추가 및 설정**
```
@EnableJpaRepositories(basePackages = "jpabook.jpashop.repository")
```
- 지정한 basePackage에서 리포지토리 인터페이스를 찾아 구현체 자동 생성
- 스프링 빈으로 등록되어 주입 가능

### 12.3 공통 인터페이스 기능

**JpaRepository 인터페이스**
- 제네릭으로 엔티티 클래스와 식별자 타입 지정
- 기본적인 CRUD 메서드 제공:
  - `save()`: 엔티티 저장
  - `findOne()`: 단건 조회
  - `findAll()`: 전체 조회
  - `delete()`: 삭제

**스프링 데이터 JPA 구현체 특징**
- `@Repository`: JPA 예외를 스프링의 추상화된 예외로 변환
- `@Transactional`: 모든 변경은 트랜잭션 내에서 실행
- `@Transactional(readOnly = true)`: 조회 메서드에 사용하여 플러시 생략 및 성능 향상

### 12.4 쿼리 메서드 기능

**3가지 쿼리 메서드 방식**

1. **메서드 이름으로 쿼리 자동 생성**
   - 규칙: `find[엔티티]By[필드][조건]`
   - 예: `findByUsername(String username)`

2. **@Query 어노테이션 사용**
   - 리포지토리에 직접 JPQL 쿼리 정의
   - 복잡한 쿼리 처리 가능

3. **네이티브 쿼리 지원**
   - `@Query(nativeQuery = true)`로 SQL 직접 작성 가능
   - 성능 최적화가 필요한 복잡한 쿼리에 사용

**파라미터 바인딩**
- 위치 기반: `?1, ?2` (권장하지 않음)
- 이름 기반: `@Param("username")` (권장)

### 12.5 명세 (Specification)

- 동적 쿼리 생성을 위한 기능
- 검색 조건이 복잡할 때 사용
- Query by Example(QBE) 등의 대안 제공

### 12.6 사용자 정의 리포지토리 구현

**커스텀 메서드 추가 방법**
- 사용자 정의 인터페이스 생성 → 구현 클래스 작성
- 리포지토리가 사용자 정의 인터페이스도 상속
- JpaRepository와 커스텀 기능 결합 가능

### 12.7 Web 확장

- Spring Data Web 확장 기능 제공
- Pageable 인터페이스로 페이징 처리
- 컨트롤러에서 페이지 객체 직접 주입 가능

### 12.8 스프링 데이터 JPA가 사용하는 구현체

- `SimpleJpaRepository`: 스프링 데이터 JPA의 기본 구현체
- 트랜잭션 관리 및 예외 변환 처리

### 12.11 정리

스프링 데이터 JPA는 데이터 접근 계층의 반복적인 코드를 제거하고 개발 생산성을 크게 향상시킴
