# 12장 과제 – 스프링 데이터 JPA

### 과제명
**Spring Data JPA Repository와 쿼리 메서드로 회원 CRUD 구현하기**

---

### 왜 이 과제를 하는가?

실무에서 JPA를 쓴다는 건 곧 **Spring Data JPA의 Repository를 쓴다**는 뜻입니다.
Entity 설계를 아무리 잘 해도, 데이터를 저장하고 조회하는 Repository를 못 만들면 아무것도 동작하지 않습니다.

이 과제는 실무에서 가장 자주 하는 작업 — **Repository 인터페이스를 만들고 쿼리 메서드를 정의하는 것** — 을 직접 경험하는 것이 목적입니다.

### 목표
- `JpaRepository`를 상속해 기본 CRUD가 자동 제공되는 것을 확인한다.
- 메서드 이름만으로 쿼리가 생성되는 **쿼리 메서드** 기능을 직접 구현한다.
- `@Query`로 JPQL을 직접 작성하는 방법을 익힌다.

### Q1. 쿼리 메서드 (메서드 이름으로 쿼리 생성)

아래 3개의 쿼리 메서드를 작성하세요.

| # | 요구사항 | 메서드 시그니처 (직접 작성) |
|---|---------|-------------------------|
| 1 | 이름이 정확히 일치하는 회원 목록 조회 | `List<Member> findBy___(String name)` |
| 2 | 이름이 일치하고, 나이가 특정 값 **초과**인 회원 목록 | `List<Member> findBy___And___(String name, int age)` |
| 3 | 이름이 특정 문자열로 **시작하는** 회원 목록 | `List<Member> findBy___(String prefix)` |

#### 힌트
- Spring Data JPA 쿼리 메서드 키워드: `findBy`, `And`, `GreaterThan`, `StartingWith`
- 메서드 이름의 필드명은 **Entity의 필드명과 정확히 일치**해야 합니다

---

### Q2. @Query를 활용한 JPQL 직접 작성

쿼리 메서드 이름만으로는 표현하기 복잡한 쿼리를 `@Query`로 작성하세요.

| # | 요구사항 | 메서드명 |
|---|---------|---------|
| 1 | 특정 **팀 이름**에 소속된 회원 목록을 조회 | `findByTeamName` |
| 2 | 이름에 특정 문자열이 **포함된** 회원 목록 조회 (LIKE) | `findByNameContaining` |

#### 힌트
```java
@Query("select m from Member m join m.team t where t.name = :teamName")
List<Member> findByTeamName(@Param("teamName") String teamName);
```
- LIKE 검색 시 JPQL에서 `%` 와일드카드를 concat으로 연결하거나 파라미터에 포함할 수 있습니다
